SEA-Nav Modified rsl_rl Integration Changes
Generated: 2026-06-04

Purpose
-------
Modified /home/hfy/SEA-Nav/training/rsl_rl so the custom SEA-Nav PPO runner can be used with Isaac Lab 2.2 / Isaac Sim 5 through a thin adapter, instead of using Isaac Lab's stock RSL-RL training script directly.


Main Changes
------------
1. Added Isaac Lab adapter
   File:
   /home/hfy/SEA-Nav/training/rsl_rl/rsl_rl/env/isaaclab_adapter.py

   New class:
   IsaacLabToSeaNavAdapter

   What it does:
   - Wraps Isaac Lab's RslRlVecEnvWrapper.
   - Converts Isaac Lab get_observations() output from:
     obs, extras
     into SEA-Nav's expected tensor-only get_observations().
   - Converts Isaac Lab step() output from:
     obs, rewards, dones, infos
     into SEA-Nav's expected:
     obs, privileged_obs, rewards, dones, infos
   - Reads critic observations from extras["observations"]["critic"] when available.
   - Falls back to policy observations when critic observations are not provided.
   - Exposes old SEA-Nav runner fields:
     num_obs
     num_privileged_obs
     rays
     num_nav_actions
     num_props
     cfg.env.his_len
   - Supports the migrated Go2 Isaac Lab task by reading rays from _rays and inferring num_props from the observation shape.
   - Caches latest observations/extras so repeated SEA-Nav getter calls do not repeatedly update Isaac Lab observation history.


2. Exported the adapter from rsl_rl.env
   File:
   /home/hfy/SEA-Nav/training/rsl_rl/rsl_rl/env/__init__.py

   Added:
   from .isaaclab_adapter import IsaacLabToSeaNavAdapter


3. Relaxed SEA-Nav VecEnv abstract interface
   File:
   /home/hfy/SEA-Nav/training/rsl_rl/rsl_rl/env/vec_env.py

   Why:
   Isaac Lab's RslRlVecEnvWrapper subclasses rsl_rl.env.VecEnv. Since the launch script puts the modified SEA-Nav rsl_rl first in sys.path, Isaac Lab sees SEA-Nav's VecEnv. The old VecEnv was an abstract class requiring get_extras() and get_privileged_observations(), so Isaac Lab's wrapper failed with:

   TypeError: Can't instantiate abstract class RslRlVecEnvWrapper with abstract methods get_extras, get_privileged_observations

   What changed:
   - Removed ABC/abstractmethod enforcement.
   - Kept the method names as a minimal interface.
   - Added default get_privileged_observations() -> None.
   - Added default get_extras() -> {}.


4. Added SEA-Nav PPO config helper
   File:
   /home/hfy/SEA-Nav/training/rsl_rl/rsl_rl/utils/sea_nav_config.py

   New function:
   get_sea_nav_train_cfg()

   What it provides:
   - SEA-Nav PPO config dictionary in the shape expected by the custom OnPolicyRunner.
   - Defaults copied from the old Go2 SEA-Nav PPO setup:
     policy_class_name = DifferentiableSafeActorCritic
     algorithm_class_name = PPO
     num_steps_per_env = 48
     max_iterations = 2000
     save_interval = 100
     experiment_name = Go2_pos_rough
     entropy_coef = 0.003
     actor_hidden_dims = [512, 256, 128]
     critic_hidden_dims = [512, 256, 128]


5. Exported the config helper from rsl_rl.utils
   File:
   /home/hfy/SEA-Nav/training/rsl_rl/rsl_rl/utils/__init__.py

   Added:
   from .sea_nav_config import get_sea_nav_train_cfg


6. Added Isaac Lab SEA-Nav training launch script
   File:
   /home/hfy/SEA-Nav/scripts/train_sea_nav_isaaclab.py

   What it does:
   - Starts Isaac Sim through isaaclab.app.AppLauncher.
   - Forces Python to import the modified local rsl_rl from:
     /home/hfy/SEA-Nav/training/rsl_rl
   - Builds the Isaac Lab task through Gymnasium.
   - Wraps the Isaac Lab env with RslRlVecEnvWrapper.
   - Wraps that with IsaacLabToSeaNavAdapter.
   - Creates the SEA-Nav custom OnPolicyRunner.
   - Saves logs under:
     /home/hfy/SEA-Nav/logs/sea_nav/<experiment_name>/<timestamp>/
   - Supports checkpoint resume through --resume, --load_run, and --checkpoint.


7. Made wandb optional in OnPolicyRunner
   File:
   /home/hfy/SEA-Nav/training/rsl_rl/rsl_rl/runners/on_policy_runner.py

   Why:
   The old runner imported wandb unconditionally. This can break Isaac Lab environments where wandb is not installed even when wandb logging is not used.

   What changed:
   - wandb import is now optional.
   - If --wandb is requested but wandb is missing, the runner raises a clear ImportError.
   - If --wandb is not requested, training can proceed without wandb installed.

How To Run
----------
Use the SEA-Nav launch script, not Isaac Lab's stock RSL-RL train.py:

/home/hfy/SEA-Nav/IsaacLab/isaaclab.sh -p /home/hfy/SEA-Nav/scripts/train_sea_nav_isaaclab.py \
  --task Isaac-Go2-Nav-Direct-v0 \
  --num_envs 1024 \
  --max_iterations 2000 \
  --headless

Known Notes
-----------
1. Full Isaac Sim training was not run during modification because it requires a live GPU/simulator session.
2. The Isaac Lab Go2 task must still be registered and available as:
   Isaac-Go2-Nav-Direct-v0
3. The migrated Go2 task currently exposes rays as _rays. The adapter supports both rays and _rays.
4. The adapter infers num_props from the flattened observation shape when the task does not expose num_props directly.
5. If you use --wandb, make sure wandb is installed in the Isaac Lab Python environment.
