SEA-Nav Go2 Obstacle Training Changes
=====================================

Date: 2026-06-11

Purpose
-------
The Go2 navigation task was upgraded from flat-plane goal navigation to an
obstacle-aware navigation environment with both static and dynamic obstacles.

The implementation is designed for scalable RL training with many parallel
environments. Obstacles are represented analytically in tensors for training
performance, and optional debug markers are available for small GUI runs.

How The New Environment Works
-----------------------------
The high-level navigation policy still outputs 3 actions:
- vx command
- vy command
- yaw-rate command

The frozen low-level locomotion policy converts these high-level commands into
12 joint actions.

Obstacle perception uses the existing ray observation channel:
- 41 rays
- 180 degree forward field of view
- robot-yaw frame
- min distance: cfg.min_ray_distance
- max distance: cfg.max_ray_distance

Each ray reports the nearest analytic obstacle distance. These ray distances are
also compatible with the existing CBF safety layer, which assumes 41 rays over a
180 degree field of view.


Reward / Termination Changes
----------------------------
The original reward terms remain:
- alive reward
- progress reward
- goal reward
- heading reward
- action-rate penalty
- joint-velocity penalty
- termination penalty

Added:
- obstacle clearance penalty
- collision penalty

Added termination:
- obstacle collision

Operational Notes
-----------------
1. Train with --headless for 1024 environments.
2. Do not use GUI training with 1024 environments on an 8 GB GPU.
3. Use GUI only for 1 to 8 environments.
4. Video recording still relies on Isaac Sim RTX rendering and may hit the
   existing librtx.scenedb crash on this machine.
5. If recording crashes, use GUI playback with --num_envs 1 and record the
   Isaac Sim window with an external screen recorder.
